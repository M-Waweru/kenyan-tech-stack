# Topic article schema

Each chapter is a **standalone topic file** under `book/part-NN/`, named `NN-slug.md`. Part intros live in the same folder as `index.md`.

## Front matter

| Field | Purpose |
|--------|---------|
| `title` | Display title, usually `N. Chapter Title` |
| `slug` | Short id (no numeric prefix), for tooling |
| `part` | Part number (1–20) |
| `part_title` | Part name for breadcrumbs |
| `chapter` | Global chapter number (1–77) |
| `topic_type` | `company` \| `rail` \| `institution` \| `infrastructure` \| `concept` |
| `period` | Rough era (e.g. `2007–today`) |
| `stack_layer` | Machine tag (e.g. `mobile_money`, `egov`) |
| `tags` | Include `topic` |

Optional later: `see_also`, `status` (for companies), `aliases`.

## Acronyms and initialisms

On the **first use in each topic file** (usually the **Lead**), spell out the **full name**, then the short form in parentheses: e.g. **Communications Authority of Kenya (CA)**. After that, the acronym alone is fine. If a term first appears late (e.g. only in **Product and mechanics**), spell it out **there** on first use. Very well-known brands (**M-PESA**) may stay as-is when no expansion is standard reading.

## Body sections (in order)

1. **Lead** — Narrative + thesis: open with a **moment**, zoom out to stakes. Match **Part I–II**: flowing prose only—no visible outline labels such as `Bridge:` or a Lead-wide `*Composite.*` (see [Structure — rule 11](structure.md#11-reader-facing-prose-not-outline-labels)). Composite vignettes belong under **`### Composite scene:`** in **History** or **Product and mechanics**.
2. **Context** — Market, regulation, distribution, prior rails.
3. **History** — Dated chronology; cite with **clickable links** to the underlying URL (not bare `SRC-XX` codes). Stable IDs remain in [Source Catalog](../../book/appendices/sources.md). See **Inline citations** below.
4. **Product and mechanics** — Flows, APIs, UX, trust model.
5. **Business model and incentives** — Who pays whom, economics.
6. **Regulation and referees** — CBK, CA, sector rules, courts.
7. **Adoption in Kenya** — Segments, geography, use cases.
8. **Ecosystem effects** — What this unlocked downstream.
9. **Setbacks and controversies** — Failures, backlash, hacks, labor.
10. **Competition and alternatives** — Landscape.
11. **Legacy and open questions**
12. **Builder read** — *Interpretation:* implications for innovators (clearly non-neutral).
13. **See also** — Links to related topic files and part index.
14. **Sources** — Bulleted list of this chapter’s URLs (may repeat links used inline). Each `SRC-XX` ID should match the catalog.
15. **Navigate** (footer) — Linear **Previous** (prior chapter or Prologue) · **Part index** · **Next** (next chapter or Cast appendix). Regenerated by `scripts/patch_topic_navigation.py` or included when you run `scripts/scaffold_topic_articles.py`.

## Inline citations (web edition)

Full step-by-step guide, template, and reference chapters: **[Web citations (linked sources)](web-citations.md)**.

- In body prose, prefer **markdown links** to the real source: e.g. `the [KBA technology timeline][kba-tech]` rather than `` (`SRC-34`) ``.
- At the **end of the topic file** (after **Navigate**, or immediately before it if your renderer is picky), add a **link reference registry**: lines like `[kba-tech]: https://…` mapping short keys to URLs. Keys need not spell out `SRC-XX`; the **Sources** section lists the official IDs.
- When several sources support one sentence, chain with **commas**, **“and”**, or **two sentences**—avoid **semicolon shopping lists** of clauses.
- For **research discipline**, seek **reputable third-party** corroboration on material claims—not only official or corporate voices. See [Web citations — Independent and third-party sources](web-citations.md#independent-and-third-party-sources-all-topics).

To expand a thin draft into full narrative with research discipline, follow **[Topic deepening workflow](topic-deepening-workflow.md)**.

## Regenerating from manuscript

After editing merged drafts in `authoring/manuscript/part-*.md`, you can refresh topic bodies from section splits:

```bash
python3 scripts/scaffold_topic_articles.py
```

That overwrites topic files: commit or backup before re-running if you have expanded sections only in `book/part-NN/` files.

To refresh only the **Navigate** footers after hand-editing a topic body:

```bash
python3 scripts/patch_topic_navigation.py
```
